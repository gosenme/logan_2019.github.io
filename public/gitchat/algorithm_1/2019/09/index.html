状态压缩问题

二进制状态压缩，是指将一个长度为 m 的 bool 数组用一个 m 位的二进制整数表示并存储的方法，主要有以下几种操作：

  • (n>> k) &1：n 在二进制下的第 k 位
  • n & ((1 << k)-1)：n 在二进制下的 0~k-1 位
  • n xor (1 << k)：n 在二进制下的第 k 位取反
  • n | (1 << k)：n 在二进制下的第 k 位赋 1
  • n & (~ (1 << k))：n 在二进制下的第 k 位赋 0

在 STL 里面，可以用 bitset 去实现，这部分很推荐大家去网上学习以下。

状态压缩常用于对一些全排列问题的优化，比如说常见的 TSP 问题，可以通过状态压缩，把复杂度优化到 O(2n×n)O(2n×n) ，这部分我将会在后续的动态规划部分详细阐述。

关于异或

异或是一种很神奇的运算，在面试中也是经常被考查到的热点。异或最神奇的性质就是一个数异或自己为 0。

利用这个性质，我们可以解决一道非常经典的面试题。

给定 N 个数，N 是奇数，在这 N 个数中，只有一个数出现 1 次，其他数都出现 2 次，求出这个数。

显然，利用异或的性质，把所有数异或一遍即可。因为相同的数，异或为 0，最后剩下的，一定是那个只出现了 1 次的数。

2 前缀和

前缀和是算法竞赛中较为常用的一种优化手段，掌握前缀和相关的知识非常重要。在这一节中我会跟大家一起学习前缀和以及差分相关的一些问题。

前缀和

首先我们先来看下面一个问题：给你一个有 N 个数组成的数组 X，其中每个数记为 X[1],X[2],X[3],…,X[N]，现在我有 m 个查询操作，每次查询会给你一个区间 (Li,Ri)(Li,Ri) ，要求你每次查询输出 ∑RiLiX∑LiRiX 。

很显然，这个题如果暴力去做的话，复杂度是 O(N×M)O(N×M) 的，那我们可以想想是否有更快的方法呢？

假设我们定义 dp[i] 表示前 i 个数的和，那对于一个区间 [L,R] 的和，是不是就可以表示成为 dp[R]-dp[L-1]，这样我们就可以在 O(M) 的时间内完成这 M 次查询操作，极大提高了效率。

前缀和是不是看上去很简单呢？没错，它就是如此简单的一个小 trick。

差分

继续上面的问题，我如果将题目改成我有 M 个修改操作。操作有 2 种，第一种，将区间 (Li,Ri)(Li,Ri) 每个数都加上 P；第二种，将区间 (Li,Ri)(Li,Ri) 每个数都减去 P。最后我只有一次查询，求 X [1],X [2],X [3],…,X [N] 的每一个数。

显然，这个题目直接用上面的知识不能直接解决了，这时候我们就需要引入差分的思想。我们会开一个 DP 数组，来储存每一次的修改操作，最后求前缀和的时候统计一下就能快速的得到正确答案了。

什么意思呢？比如说我需要对一个区间 [L,R] 中的每一个元素都加上 p，那根据差分的原理，其实我们只需要标记这一段的开始位置和这一段的结束位置，于是我们令 dp[L]+=p,dp[R+1]-=p。最后我们对 ∑i1dp∑1idp ，就可以得到修改结束后 i 位置的值

利用差分的思想就可以很方便地解决这一类相关的问题。

真题演练

3 二分

二分是一个非常基础的算法，相信大家都非常熟悉，关于二分的写法，也各人有各人的习惯，这里我就不再展开。在这里我主要跟大家讨论的是二分答案。

什么是二分答案呢？简单地说，就是我通过二分的方法，去枚举一个值，然后根据这个值和我实际要求的结果做比对，通过比对，我决定下一步是在左半区间继续去枚举，还是在右半区间继续去枚举。

4 排序

说到几大基本的排序算法及其应用场景，稳定性等问题，我相信大家背得都比我熟。但排序作为一种最最基础的算法，往往是在校招中被面试官考查得最多的。我们不仅仅应该掌握记住排序算法的复杂度，更重要的是，我们需要更深入地去理解排序算法背后的思想，以及这种思想可能产生的一些变形。

快排思想变形

说到快排，大家都非常熟悉，但不知道大家有没有考虑过这样一个问题。

给你一个无序数组 X，如何求出其中第 K 大的元素？

当然直接做排序是一种思路，复杂度是 O(NlogN) 的。但是可不可以做到更快一些呢？我们想一下逆序快排的过程，每次将一个元素固定好以后，左半区间，一定都是比它大的，而右半区间一定都是比它小的。那如果我们的基准元素位于的位置大于 K，那我们下一次是不是只需要在左半区间进行递归就行了，而不需要去递归右半区间；同理，如果小于 K，只需要递归右半区间就好了。这样的话，期望复杂度可以做到O(N)+O(2N)+O(4N)+.....<O(2×N)O(N)+O(2N)+O(4N)+.....<O(2×N) 。

6 动态规划

关于动态规划的概念，其实也不需要再赘述，百度和谷歌上已经介绍得非常详细。而动态规划，作为校招当中的热点和难点，并不像普通算法，有规律可循。它并没有啥固定的套路，更多的需要我们当时的灵感。而这种灵感，来自于平时大量的练习和总结。在这部分，我会为大家介绍一些动态规划的类型，以及一些常见的模型，同时会推荐一些比较适合入门的资料。希望能在校招中对大家有所帮助。

线性 DP

线性 DP 属于最常见的一种 DP，也是最基础的一种 DP。

线性 DP 主要就是在数组上搞，有点类似于我们高中学的数列求通项公式，只是线性 DP 通常不是一维就能记录状态的，很多时候需要多维才能记录。我们来看一些经典问题加深对线性 DP 的理解

给定一个长度 n 的数列 A，求单调递增的子序列的长度最长是多少？

这个题的状态非常好设计，令 dp[i] 表示前 i 个最长上升子序列的长度，则 dp[i] 一定有它之前比它小的数的 dp 最大值转移过来。即 dp[i]=max {dp[j],j=1,2,3…,i}，最后我们再统计最大的 dp[i] 即可。

再来看一个类似的问题：

给出两个字符串，求出这样的一个最长的公共子序列的长度

这题跟上面的题目类似，但显然一维没法记录状态，也就是我们上面提到的，需要多维数组来记录。设 dp[i][j] 表示第 1 个字符串的前 i 位和第 2 个字符串前 j 位的最长公共子序列。然后我们来看看转移。如果第一个的第 i 位和第二个的第 j 位相等，则 dp[i][j] 可由 dp[i-1][j-1] 转移过来，否则有 dp[i-1][j] 和 dp[i][j-1] 二者的最大值转移过来。

对线性 DP 状态的设计，重要的是多加练习，多加总结，一些经典的题型还包括数字三角形问题、最大子矩阵和问题，都是一样的套路，希望大家能下去搞懂。

区间 DP

与区间相关的 DP 模型也是经常被考查到的。解决这类问题，我们一般需要设计一个二维的 DP 状态，然后将问题划分为两个子问题，也就是将一段区间分成左右两个区间。最后将左右两个区间合并到整个区间。也就是我们所说的把局部最优解合并成为全局最优解。

下面我们来看这样一个问题：

给定 n 个矩阵 ｛A1,A2,…,An｝｛A1,A2,…,An｝ ，其中，AiAi 与 Ai+1Ai+1 是可乘的，(i=1,2 ,…,n-1)。用加括号的方法表示矩阵连乘的次序，不同的计算次序计算量（乘法次数）是不同的，找出一种加括号的方法，使得矩阵连乘的次数最小。

本题任务是设计一个表达式。在表达式中，一定有一个“最后一次乘法”，我们不妨设为第 k 个乘号，则在此之前已经算出了

P=A1×A2×...×Ak$，$Q=Ak+1×...×AnP=A1×A2×...×Ak$，$Q=Ak+1×...×An

显然 P 和 Q 的过程互相不干预，而且无论按照什么样的顺序，P 和 Q 的值都不会发生改变，因此只需让 P 和 Q 分别按照最优的方案计算即可。首先我们定义 dp[i][j] 为从 i 到 j 需要做多少次乘法，则根据我们上面的分析，我们知道需要把区间 (i,j) 分为两部分 (i,k) 和 (k+1,j)，我们最后的答案为这两部分的和。

最后我们就可以设计出状态转移方程了： dp[i][j]=min(dp[i][k]+dp[k+1][j]+Ai−1AkAj)dp[i][j]=min(dp[i][k]+dp[k+1][j]+Ai−1AkAj) 界是 dp[i][i]=0。最后 dp[1][n] 就是所求。

状压 DP

还记得我们在第一章当中讲过的二进制问题嘛，没错，状态压缩 DP 是利用计算机二进制的性质来描述状态的一种 DP 方式。来看这样一个问题。

给定一张 n (n≤20) 个点的带权无向图，点从 0~n-1 标号，求起点 0 到终点 n-1 的最短 Hamilton 路径。 Hamilton 路径的定义是从 0 到 n-1 不重不漏地经过每个点恰好一次。

这个题的暴力做法，大家一定都会，枚举全排列，然后判断哪一种 1~n-1 的路径是最优的，然而我们来看看数据范围 20。我们都知道，全排列的复杂度是 n!。那 20! 将是一个非常大的数，我们想想能不能优化呢？

我们是不是可以用二进制上的数代表一个点的状态，取（1）或不取（0）。题目让求从点 1 到 n 的最短汉密顿路径，即经过每个点一次，这时的状态用二进制表示就是 (1 << n)-1 （n 个 1）。用 dp[i][j] 表示在状态 i 下，从 1 到 j 的最短汉密顿路径。则 dp[i][j] 可由上一个状态（上一状态就是把 j 从当前状态中去掉）dp[i^(1 << (j-1))][k] 得到，其中保证 k 是中存在的点，即 (i>>k)&1。则状态转移方程为：dp[i][j]=min {dp [i^(1 << j)][k]+mate [k][j]}(k=1~n)，其含义就是枚举到达点 j 之前的前一个点 k，取其最短；
